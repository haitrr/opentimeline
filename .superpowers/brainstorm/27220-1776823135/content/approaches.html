3 Approaches for Manual Filter Creation

Drag on timeline → highlight on map → popover to configure

A

Dedicated "Filter Mode" Toggle

User activates a filter-drawing mode via a button in the sidebar/toolbar. While active, the timeline becomes draggable to select a range. Escape or clicking outside deactivates. Clear separation between browse mode and filter creation mode.

Pros

  • No accidental filter creation
  • Explicit intent — UI state is clear
  • Drag gesture is unambiguous in filter mode

Cons

  • Extra click to enter mode
  • Less discoverable
B

Always-On Drag to Select (Recommended)

The timeline always supports drag-to-select. A subtle "drag to filter" hint shows on hover. Dragging highlights the range immediately and shows a lightweight popover to confirm/configure. No mode switching needed.

Pros

  • Fastest path — one gesture to start
  • Live map preview as you drag
  • Most natural feel

Cons

  • Must not conflict with timeline scroll
  • Need to distinguish drag vs click clearly
C

Right-Click / Long-Press to Start Selection

User right-clicks (desktop) or long-presses (mobile) on the timeline to anchor the selection start, then drags to extend. A context menu or drag handle appears to guide the interaction.

Pros

  • No conflict with normal scroll/click
  • Works on both mobile and desktop

Cons

  • Right-click is unfamiliar in mobile apps
  • Long-press delay disrupts flow
  • Harder to discover

My recommendation: Option B — always-on drag is the fastest and most natural. The timeline scrubber already exists and the drag gesture maps cleanly to time range selection. The only risk (conflict with scroll) is solved by detecting horizontal drag distance before committing.
