Which direction matters most for PlacesPanel?

The current panel is a searchable list. Pick the direction that matters most — I'll focus the design there. You can multi-select.

A

Visual polish & hierarchy

Better typography, spacing, icons, visit-count emphasis, empty/loading states. No new features — just make the existing list look and feel better.

B

Discovery & scannability

Sort options (name / visits / recent), grouping, visit-count badge, "no results" affordance. Helps users find a place in a long list.

C

Edit/rename/delete a place directly from the list (currently requires clicking on the map). Context menu or hover actions.

D

More context per item

Last-visited date, mini breakdown, visual map dot matching panel color. Makes each row more informative at a glance.

Better empty state (onboarding illustration / CTA), loading skeleton, "no search matches" suggestion. Currently one-line grey text.
